The **TS SDK layer [SE]** is the seam I own: a small typed helper package that
wraps transaction-building, object reads, proof-payload assembly, and Walrus
reads/writes, so the frontend never hand-rolls a `Transaction`. It lives in
`scripts/` (to be promoted to a proper package).

## 1. Surface A вҖ” Provisioner UI [FE]

**Roadmap:** built in the last 3 days of Phase 2 (вүҲJune 2вҖ“4).
**Goal:** generate the agent secret, register the agent on-chain.

### Flow

1. **[FE]** Generate secret `w` client-side. Never leaves the browser/daemon.
2. **[SE/PROTO]** Compute `identity_hash = Poseidon(w)`. SE provides this as a
   TS SDK helper; the hashing primitive itself is **[TBD-1]** (see В§6).
3. **[SE]** Create the DeepBook `BalanceManager` + caps for this agent.
   Creation path is **[TBD-2]** (DeepBook app authorization вҖ” see В§6).
4. **[SE]** Build the `provision_agent` + `attach_deepbook_caps` transaction.
5. **[FE]** Sign & submit (the human owner signs this one tx вҖ” it is the only
   place a human key is involved).
6. **[FE]** Display the resulting `AgentObject` ID вҖ” this is the "robot address"
   users fund.

### SDK helper [SE] вҖ” target shape

```ts
// scripts/sdk вҖ” names provisional, will stabilize once contract sigs land
provisionAgent(params: {
  owner: string;                 // human owner address
  secretW: Uint8Array;           // generated by FE, hashed inside, never sent raw
  mandate: MandateParams;        // see Surface B
}): Promise<{ tx: Transaction }>;

// returned after execution
type ProvisionResult = {
  agentObjectId: string;         // the "robot address"
  balanceManagerId: string;
};
```

`provision_agent` parameters are **[PROTO]**-owned; the draft in
`SOLUTION_ARCHITECTURE.md` is the starting point.

## 2. Surface B вҖ” Mandate Control UI [FE]

**Roadmap:** Phase 3, concurrent with integration.
**Goal:** let the human owner set/adjust the spending policy.

### Mandate parameters (matches the `Mandate` struct draft)

| Field | Type | UI control |
|---|---|---|
| `max_amount_per_trade` | u64 | amount input (in token decimals) |
| `max_total_outflow` | u64 | amount input |
| `expiry_ms` | u64 | date/time picker вҶ’ unix ms |
| `whitelisted_pools` | set of pool IDs | multi-select of DeepBook pools |

### Integration notes

- **[SE]** provides a `listDeepBookPools()` helper returning `{ poolId, baseSymbol,
  quoteSymbol }[]` so the FE whitelist picker shows human-readable pairs, not raw
  IDs. Source: DeepBook registry / indexer.
- **[FE]** all mandate-edit txs are **owner-signed** вҖ” gate the UI behind a
  "connect owner wallet" check.
- SDK helpers [SE]: `updateMandate(...)`, `whitelistPool(...)`,
  `freezeAgent(...)`, `revokeAgent(...)` вҖ” thin wrappers over the **[PROTO]**
  contract functions.

## 3. Surface C вҖ” Real-time Feed [FE]

**Roadmap:** Phase 3, concurrent with integration.
**Goal:** show the agent's balance and a verifiable activity log.

### Data sources

| Display | Source | Owner |
|---|---|---|
| Agent balance | `BalanceManager` balances (read object / `balance<T>`) | [SE] helper |
| Trade history | DeepBook events for the `BalanceManager` ID | [SE] helper |
| Proof / intent audit trail | **Walrus** blobs written by the contract event consumer | [SE] |
| Agent status / mandate usage | `AgentObject` fields (`status`, `nonce`, `total_spent`) | [SE] helper |

### Walrus audit trail [SE]

- After each `execute_trade`, the contract emits an event. An **[SE]**-owned
  off-chain consumer writes `{ agentId, intentHash, proofHash, txDigest, ts }` to
  Walrus and stores the returned blob ID.
- **[FE]** the feed renders the Walrus blob IDs as "verifiable" links.
- Whether blob IDs are also written back on-chain is **[TBD-3]** (see В§6).

### SDK helper [SE] вҖ” target shape

```ts
getAgentFeed(agentObjectId: string): Promise<{
  status: "active" | "frozen" | "revoked";
  balances: { symbol: string; amount: string }[];
  mandateUsage: { totalSpent: string; maxTotalOutflow: string };
  activity: {
    txDigest: string;
    intentHash: string;
    walrusBlobId: string;
    timestamp: number;
  }[];
}>;
```
